Ein zentrales Problem, welches bei der Implementierung der Sampling-Solver 
aufgetreten ist, betrifft den automatischen Public Card Deal in der Game Environment.
Durch dieses Verhalten ist der gesamte Code für Sampling-Methoden fehlerhaft geworden.
[Erklärung warum das ein Problem ist, was genau schiefgeht, etc.]

Das dynamische aufbauen des Baums war meine erste Version. 
Irgendwann ist mir aufgefallen, dass die Methoden in meiner Game Environment sehr langsam sind.
Deshalb habe ich mich entschlossen den Tree im vorraus zu bauen. 
Dies brachte auch erhebliche performance Vorteile im anbracht auf die Zeit pro Iteration.
