\subsection{Monte Carlo CFR}

Die von Zinkevich et al. präsentierte Vanilla Version von CFR erfordert in jeder Iteration eine vollständige Traversierung des gesamten game trees.
Dies kann durch Sampling optimiert werden, indem immer nur ein bestimmter Teil des game trees traversiert wird.

MCCFR stellt ein allgemeines Framework für Sampling in Counterfactual Regret Minimization dar.
Ein Framework bezeichnet hier eine allgemeine Methode, die verschiedene Sampling-Strategien umfasst.
Es definiert eine Familie von Monte Carlo CFR Algorithmen, die sich darin unterscheiden, wie sie den game tree in jeder Iteration sampeln.
Der Ansatz reduziert die Kosten pro Iteration, indem er den game tree nicht vollständig traversiert und dennoch unverzerrte immediate counterfactual regrets erhält.
Dazu werden die in jeder Iteration betrachteten terminal histories eingeschränkt.

Vanilla CFR und eine Verallgemeinerung des chance sampled CFR von Zinkevich et al. sind Spezialfälle dieses Frameworks.
Zusätzlich werden zwei neue Mitglieder dieser Familie eingeführt: Outcome Sampling, bei dem nur ein einzelnes Spiel pro Iteration gesampelt wird, und External Sampling, das Chance Knoten und Aktionen des Gegners sampelt.
Unter einer angemessenen Sampling Strategie minimiert jedes Mitglied dieser Familie den overall regret und kann daher für die Berechnung von Nash-Gleichgewichten verwendet werden.

Das MCCFR Framework basiert auf der Idee, die Menge aller terminal histories $Z$ in Teilmengen zu unterteilen, die als Blöcke bezeichnet werden.
Die Definition eines Blocks hängt von der gewählten Sampling-Strategie ab.
Bei Chance Sampling CFR entspricht beispielsweise jeder Block genau einem Kartendeal, das heißt, ein Block $Q_j$ enthält alle terminal histories, die mit einer bestimmten Kartenverteilung konsistent sind.
Bei External Sampling MCCFR wird ein Block durch eine deterministische Strategie von Gegner und Zufall definiert und kann daher mehrere Kartendeals umfassen.
Bei Outcome Sampling schließlich ist jeder Block eine einzelne terminal history.

Sei $Q = \{Q_1, \ldots, Q_r\}$ eine Menge von Teilmengen von $Z$, deren Vereinigung die Menge $Z$ umfasst.
Jede dieser Teilmengen $Q_j$ wird als Block bezeichnet.
In jeder Iteration wird zufällig einer dieser Blöcke ausgewählt und nur die terminal histories in diesem Block werden betrachtet, anstatt den gesamten game tree zu traversieren.

Sei $q_j > 0$ die Wahrscheinlichkeit, dass Block $Q_j$ in der aktuellen Iteration ausgewählt wird, wobei $\sum_{j=1}^r q_j = 1$ gilt.
Da eine terminal history $z$ in mehreren Blöcken enthalten sein kann, ist die Wahrscheinlichkeit, dass $z$ in der aktuellen Iteration betrachtet wird, gegeben durch $q(z) = \sum_{j:z \in Q_j} q_j$.

Um trotz des Samplings unverzerrte Schätzer für die counterfactual values zu erhalten, wird beim Aktualisieren von Block $j$ der sampled counterfactual value wie folgt berechnet:
\begin{equation}
\tilde{v}_i(\sigma, I|j) = \sum_{z \in Q_j \cap Z_I} \frac{1}{q(z)} u_i(z) \pi_{-i}^\sigma(z[I]) \pi^\sigma(z[I], z).
\end{equation}
(\cite[Eq.~6]{lanctot2009monte})
Die Gewichtung mit $\frac{1}{q(z)}$ korrigiert dafür, dass $z$ nur mit Wahrscheinlichkeit $q(z)$ betrachtet wird.
Dieser Wert ist im Erwartungswert gleich dem counterfactual value:
\begin{equation}
\mathbb{E}_{j \sim q_j}[\tilde{v}_i(\sigma, I|j)] = v_i(\sigma, I).
\end{equation}
(\cite[Lemma~1]{lanctot2009monte})
Damit ist $\tilde{v}_i$ ein unverzerrter Schätzer von $v_i$.

Die Wahl einer Menge $Q$ zusammen mit den Sampling Wahrscheinlichkeiten $q_j$ definiert einen vollständigen sample basierten CFR Algorithmus.
Anstatt vollständige game tree Traversierungen durchzuführen, sampelt der Algorithmus einen dieser Blöcke und untersucht nur die terminal histories in diesem Block.
