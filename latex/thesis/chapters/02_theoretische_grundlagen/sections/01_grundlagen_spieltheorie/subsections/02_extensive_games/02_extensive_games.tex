\subsection{Extensive Games with Imperfect Information}
Pokerspiele sind extensive Spiele mit imperfekter Information. 
Da diese Arbeit CFR in Pokervarianten implementiert, wird dieser Spieltyp im Folgenden detailliert betrachtet.

\subsubsection{Definition}
Ein extensives Spiel mit imperfekter Information modelliert eine sequenzielle Interaktion zwischen Spielern, bei der nicht alle Spieler stets vollständig über alle vergangenen Aktionen informiert sind. 
Dabei werden die möglichen Spielverläufe als Historien erfasst, wobei eine Historie eine Abfolge von Aktionen darstellt. 
Zu jedem Zeitpunkt ist entweder ein Spieler oder der Zufall am Zug. 
Um die imperfekte Information darzustellen, werden Informationssets verwendet. 
Diese fassen Zustände zusammen, die ein Spieler nicht voneinander unterscheiden kann. 
Dies kann sowohl vergangene Aktionen anderer Spieler als auch Ausgänge von Zufallsereignissen betreffen.

Ein extensives Spiel mit imperfekter Information lässt sich formal durch folgende Komponenten definieren:
\begin{itemize}
  \item Eine Menge $H$ von Historien. 
  Eine Historie ist eine Sequenz (endlich oder unendlich) von Aktionen, getätigt von Spielern oder dem Zufall.
  Die Menge $H$ erfüllt folgende Eigenschaften:
  \begin{itemize}
    \item Die leere Sequenz $\emptyset$ ist ein Element von $H$.
    \item Ist $(a^k)_{k=1,\dots,K} \in H$ (wobei $K$ unendlich sein kann) und $L < K$, dann gilt $(a^k)_{k=1,\dots,L} \in H$.
    Das bedeutet, wenn eine Sequenz in $H$ ist, dann ist auch jeder Präfix dieser Sequenz in $H$.
    \item Wenn für eine unendliche Folge $(a^k)_{k=1}^\infty$ die Bedingung $(a^k)_{k=1,\dots,L} \in H$ für jede positive ganze Zahl $L$ erfüllt ist, dann gilt $(a^k)_{k=1}^\infty \in H$.
  \end{itemize}
  \item Eine Historie $(a^k)_{k=1,\dots,K} \in H$ ist terminal, wenn sie unendlich ist oder wenn kein $a^{K+1}$ existiert, so dass $(a^k)_{k=1,\dots,K+1} \in H$ gilt. 
  Die Menge aller terminalen Historien wird mit $Z$ bezeichnet. 
  Die Menge der Aktionen, die nach einer nicht-terminalen Historie $h$ verfügbar sind, wird mit $A(h) = \{a : (h,a) \in H\}$ bezeichnet.
  \item Die Funktion $P$ ordnet jeder nicht-terminalen Historie $h \in H \setminus Z$ einen Spieler aus $N$ oder $c$ zu. $P(h)$ ist der Spieler, der nach $h$ am Zug ist. 
  Ist $P(h) = c$, wählt der Zufall die nächste Aktion.
  \item Die Funktion $f_c$ ordnet jeder Historie $h$ mit $P(h) = c$ eine Zufallsverteilung $f_c(\cdot \mid h)$ auf $A(h)$ zu.
  $f_c(a \mid h)$ ist die Wahrscheinlichkeit, dass nach $h$ die Aktion $a$ folgt. 
  Diese Verteilungen sind unabhängig voneinander.
  \item Für jeden Spieler $i \in N$ werden die Historien $h$, bei denen $P(h) = i$ gilt, in Gruppen eingeteilt. 
  Diese Menge heißt Informationspartition $\mathcal{I}i$ von Spieler $i$. 
  Eine solche Gruppe $I_i \in \mathcal{I}_i$ heißt Informationsset von Spieler $i$. 
  Historien $h$ und $h'$, die zum selben Informationsset $I_i$ gehören, sind für Spieler $i$ nicht unterscheidbar. 
  
  Wesentlich ist dabei, dass alle Historien in einem Informationsset $I_i$ die gleichen verfügbaren Aktionen haben, also $A(h) = A(h')$ für alle $h, h' \in I_i$. 
  Für ein Informationsset $I_i$ schreiben wir $A(I_i)$ für die verfügbaren Aktionen und $P(I_i)$ für den Spieler, der am Zug ist.

  \item Für jeden Spieler $i \in N$ gibt es eine Präferenzrelation $\succeq_i$ über Lotterien auf $Z$.
  Eine Lotterie ist eine Wahrscheinlichkeitsverteilung über Endzustände.
  Es existiert eine Nutzenfunktion $u_i: Z \to \mathbb{R}$, so dass für zwei Lotterien $L_1, L_2$ gilt:
  $L_1 \succeq_i L_2 \iff \mathbb{E}[u_i \mid L_1] \ge \mathbb{E}[u_i \mid L_2]$
\end{itemize}

\subsubsection{Strategie}

Eine reine Strategie von Spieler $i \in N$ ordnet jedem Informationsset $I \in \mathcal{I}_i$ genau eine zulässige Aktion $\sigma_i(I) \in A(I)$ zu. 
Sie legt somit einen vollständigen Handlungsplan für alle Informationslagen von $i$ fest.
Eine gemischte Strategie $\alpha_i$ ist eine Wahrscheinlichkeitsverteilung über der Menge der reinen Strategien von $i$.
Eine behaviorale Strategie $\beta_i$ weist für jedes Informationsset $I \in \mathcal{I}_i$ eine Verteilung $\beta_i(\cdot \mid I)$ über $A(I)$ zu.
Der Unterschied liegt darin, dass bei einer gemischten Strategie vor Spielbeginn zufällig eine komplette reine Strategie ausgewählt wird, die dann während des gesamten Spiels deterministisch befolgt wird.
Bei einer behavioralen Strategie wird hingegen an jedem Informationsset unabhängig eine Wahrscheinlichkeitsverteilung über die verfügbaren Aktionen verwendet, sodass die zufällige Auswahl der Aktionen während des Spiels erfolgt.
Unter der Annahme von perfect recall erzeugen beide Repräsentationen dieselben Wahrscheinlichkeitsverteilungen über die Endzustände des Spiels.\footnote{\cite[Ch.~11, Sec.~11.1–11.2]{osborne1994course}}

Ein Profil behavioraler Strategien $\beta=(\beta_i)_{i\in N}$ zusammen mit den Zufallszügen $f_c$ bestimmt eine Wahrscheinlichkeitsverteilung über die terminalen Historien $Z$.
Daraus lässt sich für jede Auszahlungsfunktion $u_i$ der Erwartungswert $\mathbb{E}[u_i \mid \beta, f_c]$ berechnen.
Ein Nash‑Gleichgewicht ist ein Strategienprofil, in dem kein Spieler seinen erwarteten Nutzen durch eine einseitige Abweichung seiner Strategie verbessern kann.
Formal ist ein Strategienprofil $\sigma^*$ ein Nash‑Gleichgewicht in gemischten Strategien, wenn für jeden Spieler $i \in N$ gilt:
\begin{equation}
O(\sigma_{-i}^*, \sigma_i^*) \succeq_i O(\sigma_{-i}^*, \sigma_i)
\end{equation}
für jede gemischte Strategie $\sigma_i$ von Spieler $i$, wobei $O(\sigma)$ die durch $\sigma$ entstehende Lotterie über Endzuständen ist.
